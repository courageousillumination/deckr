In order to upload a game, you must include 6 files


  • __init__.py: This can be empty

  • config.yml: This specifies the basic rules of the game. Some attributes are game_file, game_class, max_players, min_players, zones, card_set.
    The zones key can specify a list of possible attributes including name, zone_type, owner, and multiplicity.
    The card_set key can specify a list of possible attributes including - name, front_face, back_face, card_type, kingdom_card, cost, and value.

  • layout.html: This file specifies the layout of the game which rows, zones (stacked or vertical), and more.

  • game.css: This file describes the styles that should apply to the HTML in the previous file.

  • game.js: This file describes the scripts that interact with the sockets and the DOM to coordinate moves and states.

  • [NAME].py: This file outlines the logic of the game by specifying how players, cards, zones, and game objects interact with each other. NOTE: This file must have the same name as the name of the game.
